# 制約

## 研究期間

- [確認済み事実] 前期に診断・重み・ルート生成・ノード誘導を具体化し、後期に小規模利用評価を行う計画である。
- [制約] 教材範囲、機能、評価規模を研究期間内に検証可能な範囲へ限定する。

## 実装負荷

- 全63ノードの教材・テスト・課題整備はMVP対象外。
- 高精度パーサ、AI自由記述解析、高機能IDE、管理画面、全面的技術移行は優先しない。
- UI全面改修より、ルート生成、説明可能性、保存、評価可能性を優先する。

## 被験者負担

- 長時間の二条件実験は参加者確保と離脱リスクを高める。
- 比較実験を行う場合は、課題短縮、休憩、順序効果への配慮が必要。
- 不参加・途中中止に不利益を生じさせない。

## データ・倫理

- 研究目的、収集情報、利用範囲を説明し、同意を得る。
- 氏名・学籍番号と学習ログを直接結び付けない。
- 対応表が必要なら分析データと分離する。
- 研究目的外利用、無断転載、過剰収集を行わない。
- 保存期間、削除手順、アクセス権限は評価前に確定する。

## 主張範囲

- 小規模・短期評価から長期離脱率低下を断定しない。
- 比較条件がなければ固定ルートに対する優越性を主張しない。
- 簡易エラー検出を汎用解析・自動診断と表現しない。
- 主観評価のみの場合、学習成果向上ではなく受容性・納得感・継続支援可能性として述べる。

## AIの判断範囲

AIに任せてよい:
- 資料・コードの照合
- 矛盾検出
- 仕様案、擬似コード、テスト案、Issue分解
- 文書更新案
- 確定済み仕様に基づく実装

AIが勝手に確定してはいけない:
- 研究質問の正式採用
- 被験者・人数・比較条件
- 診断重み・合格閾値の妥当性
- 倫理・個人情報管理の最終判断
- 研究成果の最終解釈
- 指導教員との合意事項
