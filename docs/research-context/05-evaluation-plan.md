# 評価計画

## 評価目的

- 個別ルートが学習者にとって納得可能か。
- 難易度と次学習内容が適切に感じられるか。
- エラー分類と復習提示が原因理解に役立つか。
- 振り返りが理解状態の把握と次の行動計画に役立つか。
- 学習継続意欲を支援する可能性があるか。

## 主な指標

### 主観指標

5件法を基本候補とする。

- ルート適合感・納得感
- 難易度の適切さ
- 次に学ぶ内容の明確さ
- 推薦理由の理解しやすさ
- エラー説明の有用性
- 復習提示の有用性
- 振り返りの有用性
- 継続意欲
- 自由記述

### 行動・結果指標

- 完了ノード数
- テスト得点・再受験回数
- 実践課題達成状況
- エラー発生・解消履歴
- 推薦ノードの選択率
- 学習時間
- 振り返り記入率

[注意] 小規模評価では統計的な優越性より、モデルが想定どおり機能するか、利用者が提示を理解・受容できるかを中心に扱う。

## 比較実験

[未確定] 固定ルートとの比較は研究質問を強化するが、必須とは確定していない。

### 案A 単群評価
- 負担: 低
- 得られる結論: 受容性・実行可能性・改善点
- 制約: 個別化の相対的優位は示せない

### 案B 被験者内比較
- 負担: 中
- 長所: 少人数でも比較可能
- リスク: 順序効果、学習内容の重複
- 対策: 条件順の入替、短い同等課題

### 案C 被験者間比較
- 負担: 高
- 長所: 条件干渉が少ない
- リスク: 必要人数が増える

## 推奨判断基準

- 参加者が10名未満または実施時間が限られる場合、単群評価か短縮被験者内比較を優先する。
- 比較を行わない場合、「固定ルートより有効」とは主張せず、「個別ルート提示モデルの実行可能性・受容性を確認した」と表現する。

## 実施手順案

1. 研究説明・同意
2. 事前属性・経験アンケート
3. 初期診断
4. ルート提示
5. 教材・確認テスト・実践課題・振り返り
6. 事後アンケート
7. 必要に応じて半構造化コメント
8. 匿名化・集計

## 未確定事項

対象者、人数、比較条件、所要時間、各質問文、合格閾値、分析手法、倫理手続の学内要件。
