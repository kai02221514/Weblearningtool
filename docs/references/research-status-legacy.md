## 2026-06-19時点の研究テーマ

初学者向けHTML/CSS学習支援において、事前診断、学習進捗、確認テスト、実践課題で検出したエラー、学習後の振り返りを統合し、学習者ごとの学習ルートの生成・更新の仕組みを研究対象としていた。

この表現は現行実装から整理した暫定テーマであり、正式な研究題目は未確定とする。

## 2026-06-19時点の研究目的

- 学習者の知識・技能・つまずきに応じて、次に学ぶべきHTML/CSSノードを説明可能な形で提示する。
- 固定順序の教材と比べ、理解度、エラーからの回復、学習効率、継続しやすさが改善するかを評価する。
- ルートを生成した根拠を保存し、研究結果を再現・検証できる状態にする。

## 2026-06-19時点の実装状況の概略

React/Vite製のプロトタイプとして、認証、初期アンケート、チュートリアル、ダッシュボード、教材、確認テスト、実践課題、振り返り、修了画面までの画面遷移が用意されている。

データ面では、[学習ノード定義](../../src/data/learningNodes.ts)にHTML 32件、CSS 31件のノードと前提関係があり、[エラーマッピング定義](../../src/data/errorMappings.ts)に14件のエラー分類がある。[MVPスコープ](./mvp-scope-initial.md)では、そのうち12ノードと8エラーを初回MVP対象として確定した。MVP対象の前提関係、エラー参照、初期推奨値、進捗IDは正規ノードIDへ揃えたが、確認テストのID・ノード対応とルート生成は未実装である。永続化は認証とプロファイル保存の一部に限られ、学習履歴はアプリのメモリ上にのみ保持される。

## 2026-06-19の確定事項

- MVP対象をHTML 7ノード、CSS 5ノードの計12ノードに限定した。
- 正規ノードIDを `^(html|css)-[0-9]{3}$` とした。
- `html-basics` 等の画面用ID、英語slug、ドット形式IDを正規ノードIDとして扱わない。
- MVP前提関係を12ノード内で閉じた。
- MVP対象エラーを8件に限定し、全参照先をMVPノードへ揃えた。
- 色指定は独立ノードにせず、`css-011` の教材内で扱うことにした。
- 確認テストIDを `quiz-{nodeId}`、中心実践課題IDを `practice-profile-card` とした。
- 現行候補表は「25件」と記載されていたが実数は26件だったため、12件採用、14件除外として訂正した。

## 機能別分類

### 実装済み

- 認証、初期アンケート、チュートリアル、学習、確認テスト、実践、振り返り、修了の画面コンポーネント
- HTML/CSS学習ノード候補と各ノードの前提関係の静的データ
- `MVP_NODE_IDS` と `getMvpLearningNodes()` によるMVP 12ノードの単一定義
- Skill / Rule / Knowledgeの区分を持つエラー分類データ
- 初期アンケートの回答スコアリングと初級・中級・上級の判定
- 固定問題による確認テストとスコア算出
- 実践課題に対する簡易的な文字列・正規表現ベースのエラー検出
- `practice-profile-card` と対象10ノードの課題メタデータ
- 振り返り入力と、選択されたつまずきに応じた簡易推奨文の生成
- Supabaseを利用したサインアップ、サインイン、プロファイル保存用エンドポイント

### 部分実装

- 推奨ノード表示: 初期値と前提判定は正規ノードIDを使うが、診断・テスト・エラー・振り返りを統合した生成結果ではない。
- ルート生成: 前提条件を満たす候補の抽出処理はあるが、順序付き学習ルート、理由コード、決定的な優先順位は未実装である。
- エラーからの推薦: MVP 8エラーは正規IDと `nodeRefs` が一致した。MVP外6エラーの旧ドット形式参照は後続移行対象である。
- 進捗管理: React state上の完了識別子を `completedNodeIds` に統一したが、初期値はデモ用で、再読み込み後には復元されない。
- 診断: 事前アンケートとレベル判定はあるが、回答を学習ノードやルート開始位置へ変換する規則はない。
- プロファイル保存: APIは存在するが、サインアップ直後のアンケートではアクセストークンがなく、実際の保存処理が呼ばれていない。

### 未実装

- `quiz-{nodeId}` に基づく確認テストID、問題ID、対応ノードIDの実装
- 診断・進捗・テスト・エラー・振り返りを統合するルート生成サービス
- ルート生成理由、ルート版、データ版を含む出力仕様
- 学習進捗、テスト試行、エラー履歴、振り返り、生成ルートの永続化と復元
- 研究評価用の事前・事後アンケート、イベント計測、データ出力
- 固定ルート等との比較条件、評価指標、対象者、実施手順の確定
- ID参照整合性、前提関係の循環、ルート順序を検証する自動テスト
