# 用語集

- **学習ノード**: 一つの学習目標、前提関係、教材、評価を持つ最小単位。
- **正規ノードID**: `html-000` 等の永続的な内部識別子。
- **MVPノード**: 初回検証対象として確定した12ノード。
- **前提ノード**: 対象ノードへ進む前に完了が必要なノード。
- **到達可能ノード**: 前提条件を満たし、未完了で、MVP集合内にあるノード。
- **開始ノード**: 診断後に最初に提示するノード。具体的な決定規則は未確定。
- **学習ルート**: 推薦順序を持つノード列と、その理由の組。
- **固定ルート**: 学習者状態によらず同じ順序で提示される比較条件。比較実験への採用は未確定。
- **個別ルート**: 学習者状態により生成・更新されるルート。
- **確認テスト**: ノード理解を確認する小テスト。IDは `quiz-{nodeId}`。
- **実践課題**: 複数ノードの知識をコード作成へ適用する課題。
- **エラー履歴**: 実践中に検出・記録された正規エラーIDと解消状態。
- **Skillエラー**: 操作、入力、手順抜けに起因する誤り。
- **Ruleエラー**: 構文・規則の誤適用。
- **Knowledgeエラー**: 役割・概念理解の不足。
- **振り返り**: 理解度、つまずき、原因、次回行動を学習者が記録する活動。
- **推薦理由**: 推薦に使った入力事実と規則を示す説明。
- **理由コード**: [暫定仕様] 推薦理由を機械可読に識別するための候補概念。具体形式は未確定。
- **ルート版**: [暫定仕様] ルート生成規則の版を識別する候補概念。必須性と形式は未確定。
- **データ版**: [暫定仕様] 学習ノード・エラーマッピング等の版を識別する候補概念。必須性と形式は未確定。
- **コード存在確認済み**: コードまたは定義の存在は確認できるが、受入条件に基づく動作検証は未完了の状態。
- **実装済み**: 正規データで動作し、受入条件に対応する検証が通った状態。
- **部分実装**: UI・処理の一部はあるが、研究仕様を満たさない状態。
- **モック**: デモ目的の固定値・仮データ・仮処理。
